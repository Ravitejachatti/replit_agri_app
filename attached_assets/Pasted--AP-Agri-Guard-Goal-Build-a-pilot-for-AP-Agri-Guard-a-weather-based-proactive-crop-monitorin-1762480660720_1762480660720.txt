

 “AP Agri Guard” 

Goal
Build a pilot for AP Agri Guard, a weather-based, proactive crop-monitoring system focused on Andhra Pradesh (Krishna & Guntur). Deliver a monorepo with:
	1.	Mobile Farmer App (React Native, Expo)
	2.	Web Dashboard (Next.js, App Router)

For now, frontend = backend: everything reads/writes local JSON + IndexedDB/AsyncStorage via a mock service layer.

Data Model (packages/shared/schema.ts)

export type District = "Krishna" | "Guntur";

export type Mandal = { id: string; name: string; district: District };
export type Village = { id: string; name: string; mandalId: string };

export type Plot = {
  id: string;
  farmerId: string;
  name: string;
  district: District;
  mandalId: string;
  villageId: string;
  centroid: { lat: number; lon: number };
  boundary?: Array<{ lat: number; lon: number }>;
  cropCycleId?: string;
};

export type Crop = "Paddy" | "Chilli";
export type Stage =
  | "Nursery" | "Transplanting" | "Tillering" | "Panicle" | "Harvest"
  | "Vegetative" | "Flowering" | "FruitSet";

export type CropCycle = {
  id: string;
  plotId: string;
  crop: Crop;
  variety?: string;
  sowingDate: string; // ISO
  stage: Stage;
  kcCurve?: Array<{ dayFromSowing: number; kc: number }>;
};

export type WxDaily = {
  id: string;
  gridId: string; date: string;
  rain_mm: number;
  tmin_c: number; tmax_c: number;
  rh_pct: number; wind_ms: number;
  radiation?: number;
};

export type SignalDaily = {
  id: string; plotId: string; date: string;
  eto: number; etc: number; vpd: number; gdd: number;
  ndvi?: number; ndwi?: number;
};

export type Hazard =
  | "PADDY_BLAST" | "PADDY_BLB"
  | "CHILLI_ANTHRACNOSE" | "CHILLI_THRIPS"
  | "FLOOD";

export type RiskEvent = {
  id: string;
  plotId: string;
  date: string;
  hazard: Hazard;
  score: number; // 0-100
  severity: "green" | "orange" | "red";
  messageKey: string; // i18n key
  explain?: string[];
  ack?: boolean;
};

export type AdvisoryTemplate = {
  id: string;
  hazard: Hazard | "GENERIC";
  messageKey: string; // i18n
  channels: Array<"IN_APP" | "WHATSAPP" | "SMS" | "IVR">;
  placeholders?: string[]; // e.g., ["plotName","mm","date"]
};

Mock Services (packages/shared/services/*)
	•	db.ts: read/write JSON in web (IndexedDB) and mobile (AsyncStorage).
	•	geo.ts: list districts/mandals/villages; return GeoJSON.
	•	plots.ts: CRUD plots (polygon + centroid); update crop/stage.
	•	signals.ts: read signals by plot/date range.
	•	alerts.ts: create/list/acknowledge alerts.
	•	Web only: Next.js /api/* routes use the same functions to simulate a backend (local JSON in packages/shared/mockdb).
	•	Mobile: calls go directly to the service layer.

YAML Rule Engine (transparent)
	•	File: packages/shared/rules.yaml with five hazards:

- id: PADDY_BLAST
  severity_thresholds: { green: 0, orange: 40, red: 70 }
  clauses:
    - all:
        - { metric: "rh_pct", op: ">=", value: 90, window: "today" }
        - { metric: "tmean_c", op: "between", value: [24, 28], window: "today" }
        - { metric: "leaf_wet_proxy", op: ">=", value: 4, window: "today" }
      weight: 30
    - any:
        - { metric: "rain_mm", op: ">=", value: 10, window: "prev24h" }
        - { metric: "dew_morning", op: "==", value: true, window: "today" }
      weight: 20
  messageKey: "hazard.PADDY_BLAST"

- id: PADDY_BLB
  severity_thresholds: { green: 0, orange: 40, red: 70 }
  clauses:
    - all:
        - { metric: "tmean_c", op: "between", value: [25, 34], window: "today" }
        - { metric: "rh_pct", op: ">", value: 70, window: "today" }
        - { metric: "wind_ms", op: ">=", value: 8, window: "prev24h" }
      weight: 35
  messageKey: "hazard.PADDY_BLB"

- id: CHILLI_ANTHRACNOSE
  severity_thresholds: { green: 0, orange: 40, red: 70 }
  clauses:
    - all:
        - { metric: "tmean_c", op: "near", value: 27, tol: 2, window: "today" }
        - { metric: "rh_pct", op: ">=", value: 80, window: "today" }
        - { metric: "rain_mm", op: ">=", value: 5, window: "prev24h" }
      weight: 30
  messageKey: "hazard.CHILLI_ANTHRACNOSE"

- id: CHILLI_THRIPS
  severity_thresholds: { green: 0, orange: 40, red: 70 }
  clauses:
    - all:
        - { metric: "vpd", op: ">=", value: 1.6, window: "today" }
        - { metric: "rain_mm", op: "<", value: 2, window: "rolling_3d" }
      weight: 30
  messageKey: "hazard.CHILLI_THRIPS"

- id: FLOOD
  severity_thresholds: { green: 0, orange: 40, red: 70 }
  clauses:
    - all:
        - { metric: "is_delta", op: "==", value: true }
        - { metric: "barrage_warning", op: "==", value: true }
        - { metric: "rain_mm", op: ">=", value: 60, window: "rolling_3d" }
      weight: 40
  messageKey: "hazard.FLOOD"

	•	Parser/engine in packages/shared/rules.ts:
	•	Converts YAML to evaluators over WxDaily + SignalDaily + crop stage.
	•	Returns { hazard, score, severity, messageKey, explain[] }.
	•	Severity thresholds: green <40, orange 40–69, red ≥70.
	•	Include explain[] listing fired clauses.
